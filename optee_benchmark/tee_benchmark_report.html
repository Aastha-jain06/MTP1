💾 Trusted Storage Performance Analysis

What We Measured:

We tested the secure storage system in the TEE across three fundamental operations:

  • WRITE: Creating new files and writing data to secure storage
  • READ: Reading existing data from secure storage
  • REWRITE: Updating existing files with new data

Each operation was tested with varying data sizes from 256 bytes to 1 MB to understand how performance scales.

Storage Speed Comparison (kB/s) - Logarithmic Scale

🔍 Key Findings:

  • Read operations are 27-40x faster than write operations (~800 kB/s vs ~20-30 kB/s)
  • Small files (<1 KB) suffer from high overhead, especially for writes
  • Optimal write performance occurs around 1-4 KB data sizes
  • Large file performance stabilizes but remains relatively slow for writes
Storage Operation Time vs Data Size

Understanding the Results:

The stark difference between read and write speeds is typical for secure storage because:

  • Writes require encryption, integrity checks, and secure flash operations
  • Reads can leverage caching and don't require the same level of security overhead
  • Write operations must ensure atomicity and data persistence in secure hardware

🔐 Cryptographic Performance Analysis

What We Measured:

We tested common cryptographic operations that secure applications use:

  • SHA-1: Legacy hash function (160-bit output)
  • SHA-224: Modern hash function (224-bit output, part of SHA-2 family)
  • AES-ECB: Electronic Codebook mode encryption (simpler, no chaining)
  • AES-CBC: Cipher Block Chaining mode (more secure, with dependency between blocks)

For each algorithm, we measured execution time and throughput across multiple iterations to assess consistency.

🔍 Key Findings:

  • SHA-224 has the lowest variance (3.63%) - most consistent performance
  • AES-CBC is ~18% slower than ECB due to block chaining overhead (expected behavior)
  • All algorithms show acceptable consistency (CV < 9%)
Performance Variability Analysis

Understanding Variability:

The Coefficient of Variation (CV) measures consistency:

  • CV < 5%: Excellent consistency (SHA-224, AES-CBC)
  • CV 5-10%: Good consistency (SHA-1, AES-ECB)
  • CV > 10%: Poor consistency (none in our tests)

Lower variance indicates more predictable performance, which is crucial for real-time secure applications.
